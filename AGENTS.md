# AGENTS.md — Flight Deal Finder

## Git Conventions

- NEVER include attribution lines (`Co-Authored-By`, `Generated by`, `🤖 Generated with`, etc.) in commit messages, PR descriptions, issue descriptions, or comments
- No footers, signatures, or branding in any git/GitHub content
- Always use `git add .` to stage all changes unless explicitly told otherwise
- Conventional commit format: `type(scope): description` — all lowercase, including proper nouns and acronyms
- Reference related GitHub issues with `fixes #N` or `closes #N` when applicable
- Commit subject lines under 72 characters
- Single commit per PR — squash before creating PR, amend additional changes to existing commit
- Always create a dedicated feature branch off upstream default branch (`git checkout -b feat/my-change upstream/main`) — never use `main` as PR head branch
- Never push directly to `main` — all changes through feature branch + PR, even hotfixes

## GitHub Issues & PRs

- Issue descriptions: short description + task checklist only. No verbose "Benefits" or "Implementation Files" sections unless explicitly requested
- PR descriptions: short summary + test plan only

## General Approach

- When requirements are ambiguous (output format, destination, tool choice, scope), ask clarifying questions BEFORE starting work
- Prefer the simplest, most direct approach — CLI tools and one-liners over scripts, direct commands over wrapper code
- Don't over-plan — for straightforward tasks, just do the work instead of presenting multi-step plans for approval
- Be concise in ALL outputs — no filler, no fluff

## Development Environment

- Always activate the Python virtual environment before running Python commands: `source venv/bin/activate`
- Install: `pip install -e ".[dev]"`
- Lint: `ruff check flight_deal_finder/`
- Test: `pytest tests/`

## Project-Specific

- API keys live in `.env` only — never committed
- Watchlist is YAML in `watchlist.yaml` — user-editable, not code-generated
- SQLite database in `data/prices.db` — auto-created, gitignored
- `cli.py` is the single entry point — `flight-deals check`, `flight-deals history`, `flight-deals watchlist`
